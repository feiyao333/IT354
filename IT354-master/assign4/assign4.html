Prototypal inheritance

1. prototypal inheritance chain: attribute

var a = {a: 1};
var b = Object.create(a); 
									
console.log(a.a); // print 1 
console.log(b.a); // print 1
b.a=5;
console.log(a.a); // print 1
console.log(b.a); // print 5
delete b.a;
console.log(a.a); // print 1
console.log(b.a); // print 1
delete a.a;
console.log(a.a); // print undefined
console.log(b.a); // print undefined

2. prototypal inheritance chain: function

let f = function () {
	this.a = 1;
	this.b = 2;
}

var o = new f(); 
									
console.log(o); 
console.log(o.a); 
console.log(o.b); 

console.log(o.prototype);
o.prototype.a = 2;
console.log(o.prototype);
o.prototype.a = 3;
console.log(o.prototype);

3. prototypal inheritance chain: this

var obj = {
	a = 1;
	b: function (){
		return this.a + 1;
	}
}

console.log(f.b()); 

var o = Object.create(obj);

when inherited function is executed, 'this' refers to the object inheriting whatever it is - not to the origina!!

4. prototypal inheritance chain: class

Microservices

There are two kinds of framework this time. The traditional one is called as Monolith and the new one is called as microservices. Comparing the two framework can help us to choose the appropriate one to save time. From my perspective, the traditional one is like the old way to build a app or system. We think whole project and prepare everything for it, which can help us to know every part in this project and keep the security. However, with the development of technology, assembly line appears in the history, which can improve the effeicient. So far, there are a lot of companies focus on build the ecosystem or evirnonment instead of the app. For instance, amazon support aws to share different service to help development focus on developing the software. And with the growing customers of ecosystem, the price will be reduced. In this case, we can find that the microservice is very flexiable and save time. On the other hand, we cannot take the responsive for the data when the customer use so that the limitation of the microservice exists. In addition, the interface with other microservice may be another question you should think about it 
								All in all, the mircoservice reduce the requirement to build app or system. However, you should pay for other stuffs. The mircoservice still is a good way to help small groups without enough equipments to acheive their goal.
